Mengenal Document Object Model (DOM)

Sebuah objek dokumen mewakili dokumen HTML yang ditampilkan dihalaman web.

Bagaimana caranya ?

Kita dapat menghubungkan antara elemen html dengan dokumen. Sehingga, kita dapat membuat seluruh komponen dari halaman html tersebut dapat diakses dan dimanipulasi. komponen tersebut seperti elemen HTML, atribut, teks, dll.

Nah, cara konten dokumen diakses dan dimanipulasi inilah yang disebut Document Object Model (DOM).

"Untuk representasi dari elemen dokumen web dalam bentuk sebuah pohon ini disebut DOM Tree".

Struktur Hierarki DOM Tree

  • Root node : node yang menjadi sumber dari diri semua lain di dalam DOM.
  • Parent node : node yang berada 1 tingkat diatas node yg lain.
  • Child node : node yang berada 1 tingkat dibawah node lain.

Contoh :

<html>
 <head>
  <title>DOM Tree</title>
 </head>
 <body>
  <h1>Hello World</h1>
  <p>Paragraf 1</p>
  <div> 
   <p>Paragraf 2</p>
  </div>
  <a href ="halaman2.html">ke Halaman 2</a> 
 </body>
</html>

Struktur Hierarki yang dihasilkan yaitu :

Jenis Node :

  • Nodelist : daftar node yang mengandung tipe node apapun.
  • HTMLCollection : daftar node yang hanya mengandung tipe node element.

HTML Collection selalu digunakan ketika akses harus diberikan ke beberapa node.

Misalnya, kebanyakan metode pemilih (seperti, getElementById) mengembalikan beberapa node atau mendapatkan referensi (sumber) ke semua anak (child). Sebuah HTMLCollection yang menyediakan metode yang sama dengan nodelist, disebut NamedItem .

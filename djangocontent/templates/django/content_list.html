Mengenal Dasar Django 1.11 LTS

Django merupakan sebuah framework powerfull untuk membuat aplikasi web dengan manggunakan bahasa pemrogramman python.

Dibuat pada tahun 2003, oleh Adrian Holovaty dan Simon Willison. Mereka awalnya hanya seorang web programmer di sebuah kantor surat kabar di Lawrence Journal-World, Kansa, US.

July tahun 2005 dirilis dan menamai Django yang berasal dari gitaris berkebangsaan Belgia-Perancis bernama Django Reinhart, karena Si andrian ngefans dengan gitaris tersebut.

Keuntungan dari Django

Object-Relation Mapping (ORM) Support

Artinya django menyediakan jembatan antara model data dan mesin database, dan mendukung serangkaian besar sistem database termasuk MySQL, Oracle, dll. Django juga mendukung NoSQL database melalui Django-nonrel fork. Untuk saat ini, hanya NoSQL database didukung oleh MongoDB dan google app engine.

Multilingual Support

Artinya Django mendukung multi-bahasa situs web melalui sistem built-in internasional. Sehingga, kamu dapat mengembangkan web atau program yang akan mendukung beberapa bahsa pemrogramman.

Framework Support

Artinya Django memilliki built-in support Ajax, RSS, Caching dan berbagai framework lainnya.

Administration GUI

Artinya django menyediakan ready-to-use user interface yang bagus untuk kegiatan administrasi.

Development Environment

Artinya django dilengkapi dengan server web ringan untuk memfasilitasi pengembangan aplikasi end-to-end dan pengujian.
Referensi : www.kapalomen.com

Alur Kerja Django

Django mendukung 2 pola alur, yaitu :

Model View Controller (MVC)

Model View Controller atau MVC adalah pola desain perangkat lunak untuk mengembangkan aplikasi web. MVC terdiri dari tiga bagian antara lain :

  • Model : lowest level dari pola yg bertanggungjawab memelihara data
  • View : bertanggungjawab untuk menampilkan semua atau sebagian data kepada user.
  • Controller : bertanggungjawab untuk merespons input user dan melakukan interaksi pada object model data.

MVC populer karena mendukung kompatibilitas aplikasi dari lapisan interface user dan support problem separation.

Model View Template (MVT)

Sedikit berbeda dengan MVC. Disini perbedaan utama antara kedua django sendiri menangani bagian Controller perangkat lunak yang mengontrol iteraksi antara model dan view, meninggalkan kita dengan template. Template disini ialah file HTML yang dicampur dengan Django Template Language (DTL).

Struktur Directory Django

Base Directory

Bisa dilihat gambar diatas tabel berwarna biru tua menunjukkan sebuah main folder dengan nama"django_project" ini disebut dengan base directory folder inilah yang akan membungkus projek django dan apps django. Untuk nama folder pada base directory dapat diubah.

Project Directory (Direktori projek)

Projek adalah kumpulan konfigurasi dan beberapa app (aplikasi) untuk suatu website tertentu.

Apps Django

Apps adalah suatu aplikasi web yang memiliki fungsi atau tugas tertentu, misal sebagai database atau sebagai aplikasi survei sederhana

static

Folder static pada django adalah folder yang berisi file-file static yang kita gunakan untuk styling, di antaranya: javascript , css , icon / svg , jquery and stuff. Kita akan menggunakan template yang yang telah tersedia.

templates

Folder templates adalah sebuah folder untuk tempat menyimpan file - file dokumen html.

db.sqlite3

Merupakan database default django. Jadi, django ini memiliki database bawaan yang bernama db.sqlite versi 3.

manage.py

Merupakan salah satu core django untuk melakukan operasi seperti migrasi database, menjalankan server, membuat apps.

Memulai Django

Ini khusus untuk pengguna sistem operasi ubuntu 16.08.

1. Install Python
  • Kunjungi website resmi python.org, download file python tersebut. Lalu extra file tersebut.
  • Buka terminal, masuk ke folder yang menyimpan file python dengan memasukkan perintah ini $ cd Desktop/ (Folder yg menyimpan file tersebut). dan install package yang dibutuhkan dengan memasukkan perintah ini $ sudo apt-get install libreadline-gplv2-dev libncursesw5-dev libssl-dev libsqlite3-dev tk-doc libgdbm-dev libc6-dev libbz2-dev.
  • Lalu masukkan beberapa perintah ini :
      • $ ./configure
      • $ make
      • $ sudo make install

    Jika kamu tiba - tiba mendapat notifikasi seperti ini :

     $ No Module Named '_ctypes'

    Lakukan install package $ libffi-dev pada synaptic package manager.

  • Setelah berhasil, silakan periksa ulang kembali dengan masukkan perintah ini :
      • $ sudo make install
      • $ which python3.7
      • $ which pip3.7
2. Buat Virtualenv Enviroment Python
  • Buat virtualenvnya dengan masukan perintah ini $ python3.7 -m venv Env (Nama virtualenv environment pythonnya).
  • Untuk aktifkan virtualenv $ source Env/bin/activate, untuk nonaktifkan virtualenv $ deactivate.
3. Install Django 1.11 LTS

Dengan cara membuat package manager python (PIP) caraya gini :

  • Periksa pip $ pip list
  • Lalu perbarui pip $ pip install --upgrade
  • Install package yang dibutuhkan $ pip install Django==1.11.*, jika sudah berhasil. Periksa kembali dengan $ pip list.
